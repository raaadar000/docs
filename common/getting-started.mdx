# Общий старт

## 1) Что нужно перед началом

- Google-аккаунт в домене компании (`@ulta.team`).
- Добавление в список деплоеров нужного проекта (делает админ).
- Доступ к ссылке FramerForge.

## 2) Базовый пользовательский сценарий

### Цель

Задеплоить актуальную версию сайта в связанный GitHub-репозиторий.

### Предпосылки

- Вы уже авторизованы в FramerForge.
- У вас есть доступ хотя бы к одному проекту.

### Шаги

1. Откройте список проектов после входа.
2. Выберите нужный проект (по названию/репозиторию).
3. Нажмите кнопку деплоя.
4. Дождитесь статуса успеха.
5. Проверьте поле "Последний деплой" (кто и когда).

### Ожидаемый результат

- Появляется успешный статус операции.
- Поле "Последний деплой" обновляется.

### Если что-то пошло не так

- Перейдите в `common/troubleshooting.mdx`.
- Если ошибка связана с правами — создайте запрос администратору.

## 3) Термины

- Проект: связка `Framer URL` + `GitHub репозиторий` + целевая папка.
- Деплоер: пользователь, которому разрешено деплоить конкретный проект.
- Админ: пользователь из `ADMIN_EMAILS`, у которого есть доступ к `/admin`.

## 4) Границы доступа

- Обычный пользователь:
  - может деплоить только выданные проекты;
  - не видит и не редактирует системные настройки.
- Администратор:
  - управляет проектами и списками деплоеров;
  - отвечает за выдачу/отзыв доступа.
