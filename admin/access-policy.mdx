# Политика доступа к админке

## Назначение

Определить, кто имеет доступ к `/admin`, по каким правилам и как происходит выдача/отзыв прав.

## Текущий список лиц с админ-доступом

`ADMIN_EMAILS_LIST`

> Заполните этот блок точным списком лиц/email, которым разрешён доступ к админке.

## Основные принципы

- Принцип минимально необходимых прав.
- Админ-доступ выдаётся только при явной операционной необходимости.
- Все изменения состава админов должны фиксироваться в согласованном канале команды.

## Процесс запроса админ-доступа

1. Инициатор оформляет запрос с обоснованием.
2. Ответственный подтверждает необходимость доступа и срок.
3. Администратор обновляет `ADMIN_EMAILS`.
4. После завершения задачи временный доступ отзывается.

## Процесс отзыва

1. Получите сигнал на отзыв доступа.
2. Удалите email из `ADMIN_EMAILS`.
3. Перезапустите/переинициализируйте сервис при необходимости.
4. Подтвердите, что пользователь больше не имеет доступа к `/admin`.
