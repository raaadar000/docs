# Операции администратора

## Зона ответственности администратора

- Управление карточками проектов.
- Управление списком деплоеров.
- Контроль корректности интеграции Framer -> GitHub.
- Поддержка и аудит выдачи доступов.

## Базовая задача 1: создать/обновить проект

### Шаги

1. Откройте админку (`/admin`).
2. Создайте новый проект или выберите существующий.
3. Заполните/обновите:
   - `Framer URL`
   - `GitHub repository`
   - `Target folder`
4. Сохраните изменения.
5. Попросите деплоера выполнить тестовый деплой.

### Ожидаемый результат

Проект доступен в рабочем интерфейсе, деплой запускается без ошибок конфигурации.

## Базовая задача 2: выдать доступ деплоеру

### Шаги

1. Откройте нужный проект в админке.
2. Добавьте пользователя в список деплоеров проекта.
3. Сохраните изменения.
4. Попросите пользователя перелогиниться и проверить наличие проекта.

### Ожидаемый результат

Пользователь видит проект и может запускать деплой.

## Базовая задача 3: отладить типовой инцидент

### Чеклист

1. Получите: проект, время ошибки, сообщение об ошибке.
2. Проверьте настройки проекта в админке.
3. Сверьте, что пользователь есть в нужных списках доступа.
4. При необходимости подключите разработчика для анализа логов/кода.

### Ожидаемый результат

Инцидент локализован как проблема прав, конфигурации или кода.

## Безопасная эксплуатация

- Не храните секреты в документации и чатах.
- Минимизируйте количество администраторов.
- Проводите периодический ревью списка `ADMIN_EMAILS`.
- Фиксируйте значимые изменения прав и конфигурации.
