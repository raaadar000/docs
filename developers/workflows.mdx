# Гайд для разработчиков

## Цель роли

Поддерживать инфраструктуру FramerForge, обеспечивать стабильный деплой и быстро устранять технические инциденты.

## Базовая задача 1: локальный запуск

### Предпосылки

- Node.js 20+.
- Доступ к значениям переменных окружения.

### Шаги

1. Создайте `.env` на основе `.env.example`.
2. Заполните ключевые переменные (`ENCRYPTION_KEY`, `ADMIN_EMAILS`, `NEXTAUTH_*`, `GOOGLE_*`).
3. Выполните `npm ci`.
4. Примените миграции (`npx prisma migrate dev` или `npx prisma migrate deploy`).
5. Запустите `npm run dev`.

### Ожидаемый результат

Приложение открывается локально, вход через Google работает.

## Базовая задача 2: деплой через Docker

### Шаги

1. Проверьте корректность переменных окружения в окружении деплоя.
2. Запустите `docker compose up -d --build`.
3. Проверьте доступность приложения по host/port.
4. Проверьте логи контейнера при необходимости (`docker compose logs -f app`).

### Ожидаемый результат

Контейнер запущен, миграции применены, приложение отвечает.

## Базовая задача 3: управление инцидентами

### Типовые кейсы

- Ошибки OAuth / входа пользователей.
- Ошибки деплоя конкретного проекта.
- Проблемы с CMS-контентом после экспорта.

### Минимальный протокол диагностики

1. Зафиксируйте симптомы, время и affected project.
2. Проверьте воспроизводимость.
3. Проверьте server logs и runtime ошибки в браузере.
4. Оцените, это проблема доступа, конфигурации или кода.

### Результат

Инцидент локализован, назначены владелец и следующий шаг.

## Работа с админ-доступом

- Изменение списка админов выполняется только через `ADMIN_EMAILS`.
- Обычным пользователям админка недоступна по дизайну.
- Любое расширение прав должно быть согласовано и задокументировано.
