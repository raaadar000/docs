# Гайд для product manager

## Цель роли

Организовать предсказуемый релиз: согласовать готовность, инициировать деплой, проверить результат и эскалировать инциденты.

## Базовая задача 1: подготовка релиза

### Предпосылки

- Есть согласованный scope релиза.
- Дизайн и контент готовы.

### Шаги

1. Подтвердите у дизайнера и разработчика готовность изменений.
2. Проверьте, что выбран корректный проект FramerForge.
3. Запланируйте окно релиза и ответственных.

### Ожидаемый результат

Релиз можно запускать без блокеров.

## Базовая задача 2: контроль деплоя

### Шаги

1. Запустите деплой (или попросите назначенного деплоера).
2. Дождитесь статуса выполнения.
3. Сверьте "Последний деплой" и время операции.
4. Проведите smoke-check основных пользовательских сценариев.

### Ожидаемый результат

Деплой успешный, smoke-check пройден, релиз подтверждён.

## Базовая задача 3: эскалация проблем

### Когда эскалировать

- Деплой падает повторно.
- Страница открывается, но контент отображается некорректно.
- Нужны изменения прав доступа.

### Кому эскалировать

- Администратору: права, настройки проекта, деплоеры.
- Разработчику: технические ошибки, runtime, логи.

### Какие данные приложить

- Название проекта.
- Время и шаг, на котором произошла ошибка.
- Текст ошибки/скриншот.

## Ограничения по админке

PM может не иметь доступа к `/admin`. Для административных действий используйте процесс запроса к администратору.
